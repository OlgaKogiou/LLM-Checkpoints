# CheckFreq — Docker Quickstart

These are **copy‑paste** setup + run instructions that worked on our V100 box. Drop this into your repo’s README if useful.

---

## 0) Prerequisites

* Linux host with NVIDIA driver installed (e.g. 525+). Verify:

  ```bash
  nvidia-smi
  ```
* Docker and NVIDIA Container Toolkit configured (so `--gpus all` works):

  ```bash
  docker --version
  docker run --rm --gpus all nvidia/cuda:11.8.0-base-ubuntu22.04 nvidia-smi
  ```
* A dataset in ImageNet-like layout (or Imagenette) on the host, e.g.:

  ```
  ~/checkfreq_env/imagenet/
  ├── train/
  └── val/
  ```

> **Why these versions?** We use the **legacy, known‑good** NVIDIA PyTorch container `nvcr.io/nvidia/pytorch:19.05-py3` (PyTorch 1.1/CUDA 10.1). It works out of the box with CheckFreq’s code paths we’re using (native dataloader, no DALI).

---

## 1) Pull code & prepare folders

```bash
mkdir -p ~/checkfreq_env && cd ~/checkfreq_env

# Clone your repos here (adjust URLs/branches to your fork)
# CheckFreq repo
git clone <YOUR_CHECKFREQ_GIT_URL> CheckFreq
# CoorDL repo (if needed by your training script)
git clone <YOUR_COORDL_GIT_URL> CoorDL

# Put / link your dataset here
# e.g., ln -s /mnt/datasets/imagenet ~/checkfreq_env/imagenet
```

---

## 2) Start a container (with GPUs and volumes)

```bash
docker run --rm -it --gpus all --ipc=host \
  -v ~/checkfreq_env:/work \
  nvcr.io/nvidia/pytorch:19.05-py3 bash
```

Notes:

* `--ipc=host` avoids PyTorch shared‑memory warnings.
* If you want to pin to specific host GPUs at start, use:

  ```bash
  docker run --rm -it --gpus '"device=1,2,3"' --ipc=host \
    -v ~/checkfreq_env:/work \
    nvcr.io/nvidia/pytorch:19.05-py3 bash
  ```

---

## 3) In the container: set paths & sanity‑check CUDA

```bash
# Inside container
export PYTHONPATH=/work/CoorDL:/work/CheckFreq:$PYTHONPATH
cd /work/CheckFreq

# Optional: quick GPU sanity
python - <<'PY'
import torch
print('torch', torch.__version__, 'cuda', torch.version.cuda)
print('visible gpus:', torch.cuda.device_count())
for i in range(torch.cuda.device_count()):
    print(i, torch.cuda.get_device_name(i))
PY
```

---

## 4) Run training (native dataloader)

We avoid DALI here (older DALI ops differ). Use the **native dataloader** path.

### A) Single‑GPU example

```bash
export CUDA_VISIBLE_DEVICES=1    # pick a free GPU (avoid a busy GPU 0 if needed)
python -m torch.distributed.launch --nproc_per_node=1 \
  models/image_classification/pytorch-imagenet-cf.py \
  -a resnet18 -b 128 --workers 2 --epochs 1 \
  --deterministic --noeval --barrier \
  --checkfreq --chk-prefix ./chk_smoke/ \
  --data /work/imagenet | tee /work/run_checkfreq_1gpu.log
```

### B) Multi‑GPU example (3 GPUs, skipping GPU 0)

```bash
export CUDA_VISIBLE_DEVICES=1,2,3
python -m torch.distributed.launch --nproc_per_node=3 \
  models/image_classification/pytorch-imagenet-cf.py \
  -a resnet18 -b 128 --workers 3 --epochs 1 \
  --deterministic --noeval --barrier \
  --checkfreq --chk-prefix ./chk_smoke/ \
  --data /work/imagenet | tee /work/run_checkfreq_3gpu.log
```

> **Tip (OOM):** If you hit CUDA OOM, drop `-b` (batch size), e.g. `-b 64` or `-b 32`, and/or reduce `--workers`.

---

## 5) Resuming / Checkpoints

CheckFreq uses a checkpoint prefix directory. The above commands write to `./chk_smoke/` under `/work/CheckFreq`. To resume a broken run, just rerun with the same `--chk-prefix` and it will pick up the latest consistent state (no manual `--resume` path required).

## 6) Verifying local Docker images **without pulling**

```bash
# List images you already have
docker images

# Run a quick nvidia-smi using local image only (no network pull)
for img in \
  nvidia/cuda:11.8.0-base-ubuntu22.04 \
  nvidia/cuda:12.4.0-base-ubuntu22.04 \
  nvcr.io/nvidia/pytorch:19.05-py3
Do
  echo ">>> $img"
  docker image inspect "$img" >/dev/null 2>&1 \
    && docker run --rm --pull=never --gpus all "$img" nvidia-smi >/dev/null \
    && echo "ok: present & runnable" \
    || echo "miss: not present or not runnable without pull"
Done

# Show metadata
docker image inspect nvcr.io/nvidia/pytorch:19.05-py3 \
  --format 'RepoTags: {{.RepoTags}} | Id: {{.Id}} | Created: {{.Created}} | Size(bytes): {{.Size}}'
```

## 7) Common issues

* **Another user is on GPU 0** → mask it out:

  ```bash
  export CUDA_VISIBLE_DEVICES=1,2,3
  ```
* **CUDA OOM** → lower per‑GPU batch size: `-b 64` (or `-b 32`).
* **Slow dataloading** → reduce workers (`--workers 2`) or ensure dataset is on fast storage.
* **DALI errors** (e.g. missing ops) → keep `--dali` **off** with this legacy container.
* **Logs** → we use `tee` to write logs under `/work/*.log`.
